# Vibe Broker — Backlog

## B1: Отправка сообщений брокеру от имени РОПа

**Идея:** Скилл `/rop` сам отправляет Telegram-сводку брокеру, без copy-paste.

**Что нужно:**
- Telegram Bot API token (бот, от имени которого слать)
- Telegram user_id каждого брокера (или username)
- Маппинг: Kommo user_id → Telegram chat_id

**Реализация:** `POST https://api.telegram.org/bot{TOKEN}/sendMessage` с `chat_id` и `text`.

**Блокер:** Нужен Telegram bot token + таблица broker → tg_chat_id.

## B2: Скилл передачи базы в работу менеджеру

**Идея:** `/handoff <менеджер> [фильтры]` — передаёт пачку лидов конкретному брокеру с полной настройкой.

**Включает:**
1. Фильтрация лидов (по этапу, тегу, дате создания, источнику)
2. Обогащение — подтягивает данные о лиде (контакт, история, последнее событие)
3. Смена ответственного (`PATCH /api/v4/leads` → `responsible_user_id`)
4. Постановка задачи на первый контакт (`POST /api/v4/tasks`)
5. Движение по этапам — перевод в нужный статус (`status_id`)
6. Отчёт: что передано, кому, сколько лидов

**Пример вызова:**
```
/handoff Sergey Borisov stage=Handed_To_Work days_since_created<30
```

**Блокер:** Нужно определить workflow — какие этапы считаются "готовыми к передаче".

## B3: Автоматический дейли дайджест РОПу

**Идея:** Каждое утро в 9:00 — отчёт по всем воронкам в Telegram.

**Реализация:** Scheduled trigger (cron) → запуск `/rop` → отправка в TG.

---

## B4: Скилл `/today` — что делать брокеру сегодня

**Идея:** Показывает брокеру его лиды с просрочками, задачи на сегодня из CRM, приоритеты.

**Источники:** Kommo tasks (due_date = today) + leads inactive >7d.
