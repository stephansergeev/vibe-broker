# /rop — РОП-анализатор воронки Sell My Boat

Анализируй воронку "Sell My Boat" в Kommo CRM и выдавай отчёт: кто из менеджеров недорабатывает и где.

## Алгоритм

1. **Загрузи переменные** из `.env` в корне репозитория:
   ```bash
   source .env 2>/dev/null || source $(git rev-parse --show-toplevel)/.env
   ```

2. **Получи всех менеджеров** (users):
   ```bash
   curl -s -H "Authorization: Bearer $KOMMO_ACCESS_TOKEN" \
     "$KOMMO_BASE_URL/api/v4/users?limit=250"
   ```
   Построй словарь id → имя.

3. **Получи все открытые лиды** воронки Sell My Boat (постранично):
   ```bash
   curl -s -H "Authorization: Bearer $KOMMO_ACCESS_TOKEN" \
     "$KOMMO_BASE_URL/api/v4/leads?pipeline_id=$KOMMO_SMB_PIPELINE_ID&status_id=0&limit=250&page=1&with=contacts"
   ```
   Статус 0 = все открытые. Листай страницы пока не кончатся лиды.

4. **Для каждого лида** вычисли:
   - Менеджер (`responsible_user_id`)
   - Этап воронки (`status_id` → название)
   - Дней без активности = (today - `updated_at`) в днях
   - Дней с момента создания = (today - `created_at`) в днях

5. **Получи последние события** для лидов с inactive_days > 7:
   ```bash
   curl -s -H "Authorization: Bearer $KOMMO_ACCESS_TOKEN" \
     "$KOMMO_BASE_URL/api/v4/events?filter[entity_type]=lead&filter[entity_id]={id}&limit=1"
   ```
   Определи тип последнего события и когда было.

6. **Выяви проблемные паттерны** по каждому менеджеру:
   - Лиды без активности > 7 дней (требуют внимания)
   - Лиды без активности > 14 дней (критично)
   - Лиды без активности > 30 дней (фактически заброшены)
   - Лиды на одном этапе > 14 дней (застряли)

## Формат отчёта

Выводи в следующем порядке:

### 🔴 Итог по менеджерам (таблица)

| Менеджер | Всего лидов | >7д без актив. | >14д без актив. | >30д заброшены |
|----------|-------------|----------------|-----------------|----------------|
| Имя      | N           | N              | N               | N              |

### 🔶 Критичные лиды (>14 дней без активности)

Для каждого лида:
```
[ID] Название лида — Менеджер
  Этап: X | Дней без активности: N | Создан: N дней назад
  Последнее событие: {тип события} {дата}
  🔗 ссылка
```

Сортируй по убыванию дней без активности.

### ✅ Рекомендации

На основе анализа дай 3-5 конкретных рекомендаций РОПу:
- Кому позвонить/написать сегодня
- Каких менеджеров нужно проверить отдельно
- Какие лиды стоит закрыть как нецелевые

## Важно

- Сегодняшняя дата: используй `date +%s` для unix-timestamp сравнений
- Kommo `updated_at` — unix timestamp
- Пагинация: если в ответе есть `_links.next`, запрашивай следующую страницу
- Не показывай закрытые лиды (статус "Закрыто - нецелевой" или "Успешно завершено")
